**CVOYA Graph 1.0 is the first stable release of CVOYA's type-safe graph library for .NET 10.**
It provides one provider-neutral model for graph data, LINQ queries, traversal, transactions,
set-based mutation, serialization, and provider certification.

## What 1.0 delivers

- An identity-free domain model: `IEntity`, `Node`, and `Relationship` do not impose IDs,
  relationship endpoints, or direction fields. Domain keys are optional and may be composite.
- Neo4j, PostgreSQL + Apache AGE, and in-memory providers behind the same public API.
- Typed and dynamic queries, path segments and paths, full-text search, grouped aggregation,
  streaming terminals, atomic relationship creation, and set-based updates and deletes.
- Incremental serialization code generation, compile-time model analyzers, and a packaged provider
  compatibility suite for third-party implementations.
- Native graph storage interoperability: compatible Neo4j and AGE rows can be queried without
  CVOYA-specific identity or discriminator metadata.

## Provider notes

Neo4j is the complete database-backed reference provider. Apache AGE 1.7 supports the shared v1
contract except nested transactions, whole-entity ordering, and shortest path; those differences
are capability-gated and documented rather than emulated with partial fallbacks. The in-memory
provider is a deterministic application test double and executable provider reference, not a
production store.

The Apache AGE provider includes work adapted from [@paule96's contribution in
PR #66](https://github.com/cvoya-com/graph/pull/66).

## Moving from a preview release

The stable packages use the `Cvoya.Graph.*` IDs and replace the earlier
`Cvoya.Graph.Model.*` package line. The v1 storage model deliberately does not read or rewrite
alpha-era universal IDs, relationship endpoint fields, root labels, or provider metadata. Back up
pre-v1 data, update the application model, then **recreate and reimport** the database or perform
your own reviewed transformation. See the
[v1 migration guide](https://github.com/cvoya-com/graph/blob/v1.0.0/docs/migration-0.x.md).

## Install and learn

Install the provider for your database from the
[CVOYA NuGet profile](https://www.nuget.org/profiles/cvoya). The tagged source archive
(`cvoya-graph-source.zip`) is attached below. Start with the
[quick start](https://github.com/cvoya-com/graph/blob/v1.0.0/README.md#quick-start), then use the
[complete documentation](https://oss.cvoya.com/graph/) for querying, transactions, provider
behavior, migration, and extension guidance.
